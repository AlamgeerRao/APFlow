# AP Flow — Definition of Done

**Status:** Approved — Permanent Reference
**Audience:** All AI development agents and human engineers
**Purpose:** This document defines the mandatory Definition of Done for every work package. No work package may be marked complete until every item below is satisfied.

---

## Business

- [ ] Requirement implemented.
- [ ] Acceptance criteria satisfied.

## Code

- [ ] Builds successfully.
- [ ] No compiler warnings.
- [ ] No unused code.
- [ ] Follows coding standards.

## Testing

- [ ] Unit tests added.
- [ ] Existing tests pass.
- [ ] Edge cases considered.

## Architecture

- [ ] Follows approved architecture.
- [ ] No unauthorised dependencies.
- [ ] No duplicated logic.

## Security

- [ ] Input validation completed.
- [ ] Secrets protected.
- [ ] Logging appropriate.

## Documentation

- [ ] Public interfaces documented where required.
- [ ] README updated if necessary.

## Quality

- [ ] Peer review completed.
- [ ] QA review completed.
- [ ] Ready for Git commit.

---

## AI Agent Rules

No work package is complete until every checklist item above has been satisfied.
